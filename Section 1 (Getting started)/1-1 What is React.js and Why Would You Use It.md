## React là gì và tại sao nên sử dụng?

## Định nghĩa React

- React là một **thư viện JavaScript** (JavaScript library) dùng để xây dựng giao diện người dùng (user interfaces)[maybe+2](https://maybe.works/blogs/react-js-advantages)​
    
- Theo trang chính thức react.dev: React là "thư viện dành cho giao diện web và native" (library for web and native user interfaces)[netguru+1](https://www.netguru.com/blog/why-use-react)​
    
- Đây là thư viện mã nguồn mở, được phát triển và duy trì bởi Meta (Facebook)[geeksforgeeks+1](https://www.geeksforgeeks.org/reactjs/what-are-the-advantages-of-react-js/)​
    

## Tại sao nên sử dụng React?

## Trải nghiệm người dùng mượt mà

- Các trang web sử dụng React như **Netflix** có khả năng chuyển đổi giữa các trang cực kỳ mượt mà và gần như tức thì[advanceecomsolutions](https://www.advanceecomsolutions.com/insights/blog/benefits-of-react-javascript-library/)​
    
- Không có hiệu ứng "đợi tải trang mới" từ server, tạo cảm giác giống như ứng dụng di động[influize+1](https://www.influize.com/blog/benefits-react-js)​
    
- JavaScript chạy ngầm trong trình duyệt có thể cập nhật trang và giao diện người dùng mà không cần tải lại trang[maybe+1](https://maybe.works/blogs/react-js-advantages)​
    

## Cơ chế hoạt động

- JavaScript có khả năng đọc và thao tác với trang web sau khi nó đã được tải[dev+1](https://dev.to/mourya_modugula/a-comprehensive-guide-to-reactjs-the-popular-javascript-library-for-building-user-interfaces-d9e)​
    
- Khi người dùng tương tác (ví dụ click vào tab phim), React tự động lấy dữ liệu ở background mà không rời khỏi trang[netguru+1](https://www.netguru.com/blog/why-use-react)​
    
- React sau đó cập nhật những gì hiển thị trên màn hình với dữ liệu mới, tạo chuyển tiếp mượt mà[geeksforgeeks+1](https://www.geeksforgeeks.org/reactjs/what-are-the-advantages-of-react-js/)​
    

## Tại sao không dùng JavaScript thuần?

- Làm việc với **vanilla JavaScript** (JavaScript thuần, không dùng thư viện) tất nhiên là một lựa chọn, nhưng không phải là lựa chọn tốt[dev+1](https://dev.to/mourya_modugula/a-comprehensive-guide-to-reactjs-the-popular-javascript-library-for-building-user-interfaces-d9e)​
    
- JavaScript thuần có những nhược điểm:
    
    - **Cồng kềnh** (cumbersome): khó sử dụng và tốn nhiều công sức hơn[influize+1](https://www.influize.com/blog/benefits-react-js)​
        
    - **Dễ gây lỗi** (error-prone): tăng nguy cơ phát sinh bug[claritee+1](https://claritee.io/blog/what-is-react-a-comprehensive-introduction-to-the-popular-javascript-library/)​
        
    - **Không thực tế** cho ứng dụng phức tạp: đặc biệt với các giao diện người dùng phức tạp như Netflix, việc dùng JavaScript thuần sẽ là "cực kỳ khó khăn" và không phải là lựa chọn khả thi[maybe+1](https://maybe.works/blogs/react-js-advantages)​
        

## Lợi ích khi sử dụng React

- **Virtual DOM**: React sử dụng DOM ảo (Virtual DOM) để tối ưu hóa hiệu suất, chỉ cập nhật những phần thay đổi thay vì toàn bộ trang[geeksforgeeks+2](https://www.geeksforgeeks.org/reactjs/what-are-the-advantages-of-react-js/)​
    
- **Kiến trúc component**: cho phép tạo các thành phần UI có thể tái sử dụng, giúp phát triển nhanh hơn và dễ bảo trì hơn[dev+2](https://dev.to/mourya_modugula/a-comprehensive-guide-to-reactjs-the-popular-javascript-library-for-building-user-interfaces-d9e)​
    
- **Hiệu suất cao**: giảm thiểu thao tác trực tiếp với DOM thật, tăng tốc độ render và cải thiện trải nghiệm người dùng[influize+2](https://www.influize.com/blog/benefits-react-js)​
    
- **Linh hoạt và module hóa**: kiến trúc linh hoạt giúp quản lý các ứng dụng lớn và phức tạp dễ dàng hơn[netguru+1](https://www.netguru.com/blog/why-use-react)​
